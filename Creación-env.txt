# ¿Qué es un venv? (Virtual Environment o Entorno Virtual)

# Es básicamente un "espacio aislado" donde puedes instalar
# librerías de python sin afectar a tu sistema global ni a otros proyectos. 

# Piensa en ello como una caja independiente para tu proyecto:
# dentro de esa caja puedes tener versiones específicas de Python y librerías,
# y no va a chocar con otros proyectos que tengas en tu computador. 
"Los entornos virtuales pueden romper el git bash, entonces se trabajan en la terminal del pc" 

# 1. Creas un venv dentro de tu proyecto
python -m venv venv (el último venv es el nombre, usalmente se deja así). 

# 2. Activas el venv
# En Windows:
venv\Scripts\activate
# En Mac/Linux:
source venv/bin/activate 
#Una vez activado antes de la ruta dice (venv)

# Ahora, cualquier librería que instales con pip install solo se instala 
# dentro del venv y no en el sistema global

# 3. Salir del venv
deactivate 

# Para instalar librerias
pip install nombre_libreria (nosotras instalamos flask -> pip install flask) 

# Crea archivo requirements.txt para dejar todas las librerías a usar en el proyecto que estén instaladas
pip freeze > requirements.txt (según google: is used in Python development to generate a requirements.txt file, which lists all the Python packages installed in the current environment along with their exact version numbers.) 

# Correr el siguiente comando para instalar
pip install -r requirements.txt (para instalar las librerías de ese archivo)
